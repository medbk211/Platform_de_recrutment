# Identification des besoins fonctionnels - Plateforme de recrutement

## 1. Objectif du document

Ce document presente les besoins fonctionnels essentiels de la plateforme de recrutement en ligne. Il structure les attentes metier autour de quatre axes principaux :

- gestion des comptes ;
- espace candidat ;
- espace recruteur ;
- espace administrateur.

L'objectif est de definir clairement les fonctions que le systeme doit offrir aux differents utilisateurs.

## 2. Gestion des comptes

La plateforme doit permettre :

- l'inscription des utilisateurs avec choix du role : candidat ou recruteur ;
- l'authentification securisee avec connexion et deconnexion ;
- la recuperation et la reinitialisation du mot de passe ;
- la gestion du profil utilisateur : nom, prenom, email, telephone, photo, mot de passe ;
- la verification ou validation des comptes recruteurs si necessaire ;
- la gestion des droits d'acces selon le role ;
- la desactivation ou suppression d'un compte ;
- la conservation d'un historique minimal des actions importantes liees au compte.

## 3. Besoins fonctionnels du Candidat

Le candidat doit pouvoir :

- creer et completer son profil professionnel ;
- renseigner ses competences, experiences, formations et informations personnelles ;
- televerser un CV, une lettre de motivation et d'autres documents utiles ;
- consulter la liste des offres d'emploi disponibles ;
- rechercher des offres par mots-cles ;
- filtrer les offres selon plusieurs criteres : secteur, localisation, type de contrat, salaire, experience ;
- consulter le detail d'une offre ;
- postuler a une offre directement en ligne ;
- suivre l'etat de ses candidatures ;
- consulter l'historique de ses candidatures ;
- recevoir des notifications lors d'un changement de statut ;
- echanger avec le recruteur via une messagerie interne ;
- consulter les convocations et dates d'entretien.

## 4. Besoins fonctionnels du Recruteur

Le recruteur doit pouvoir :

- creer et gerer le profil de son entreprise ;
- publier une offre d'emploi ;
- modifier, archiver ou supprimer une offre ;
- consulter la liste de ses offres publiees ;
- consulter les candidatures recues pour chaque offre ;
- filtrer, trier et rechercher les candidatures ;
- consulter le profil detaille d'un candidat ;
- telecharger ou visualiser les CV et documents joints ;
- faire evoluer le statut d'une candidature ;
- envoyer des messages aux candidats ;
- planifier des entretiens ;
- suivre l'avancement des recrutements a partir d'un tableau de bord ;
- consulter des indicateurs simples : nombre d'offres, nombre de candidatures, statuts en cours.

## 5. Besoins fonctionnels de l'Administrateur

L'administrateur doit pouvoir :

- gerer les comptes utilisateurs ;
- activer, suspendre ou supprimer un compte ;
- controler et valider les comptes recruteurs ;
- moderer les offres publiees ;
- supprimer ou corriger les contenus non conformes ;
- superviser l'activite globale de la plateforme ;
- gerer les categories et referentiels : secteurs, types de contrat, niveaux d'experience ;
- consulter les statistiques globales de la plateforme ;
- acceder a un journal des actions importantes ;
- parametrer certains elements generaux du systeme.

## 6. Priorites fonctionnelles pour un MVP

Les fonctionnalites prioritaires pour une premiere version sont :

1. inscription et authentification ;
2. gestion des comptes et des roles ;
3. profil candidat et depot de CV ;
4. publication et gestion des offres d'emploi ;
5. recherche et filtrage des offres ;
6. depot de candidature ;
7. suivi du statut des candidatures ;
8. tableau de bord recruteur ;
9. administration de base des utilisateurs et des offres.

## 7. Conclusion

Ces besoins fonctionnels constituent le socle du projet de plateforme de recrutement. Ils couvrent les attentes principales des candidats, des recruteurs et de l'administrateur, tout en definissant un perimetre clair pour la conception du systeme et la priorisation du developpement.
